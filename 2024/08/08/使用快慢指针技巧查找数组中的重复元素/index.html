使用快慢指针技巧查找数组中的重复元素

在算法面试中，有一个经典问题是查找数组中的重复元素。特别是当数组中的每个元素都在 [1, n] 范围内，并且只有一个重复的元素时，如何在不修改数组且只使用常数级别的额外空间的情况下找到这个重复的元素？本文将介绍一种高效的方法——快慢指针（Floyd’s Tortoise and Hare）技巧，这种方法通常用于检测链表中的环。

问题描述

给定一个包含 n + 1 个整数的数组 nums，其元素取值范围为 [1, n]，其中只有一个重复的整数，但它可能重复多次。我们的任务是找到这个重复的整数。

快慢指针技巧简介

快慢指针（Floyd’s Tortoise and Hare）是一种巧妙的算法，常用于检测链表中的环。通过将数组的元素值视为链表中的指针，我们可以将这个问题转化为检测链表中的环问题。

方法步骤

  1. 初始化快慢指针：两个指针 slow 和 fast，初始时 slow 从数组的第一个元素开始，fast 从第二个元素开始（即 nums[0]）。

  2. 寻找相遇点：快指针每次移动两步，慢指针每次移动一步，直到两者相遇。相遇点在环中。

  3. 寻找环的入口：将快指针重置到起点，然后快慢指针同步移动，每次移动一步，直到两者再次相遇。相遇点即为重复元素。

代码实现

以下是实现这一算法的 Java 代码：

class Solution {
    public int findDuplicate(int[] nums) {
        int slow = 0;
        int fast = nums[0];

        // 使用快慢指针查找相遇点
        while (slow != fast) {
            fast = nums[nums[fast]]; // fast 每次走两步
            slow = nums[slow];       // slow 每次走一步
        }

        // 相遇之后，fast 位于 slow 的下一个位置，slow 回到原点
        fast = nums[slow];
        slow = 0;

        // 同步步调前进，直到再次相遇
        while (slow != fast) {
            fast = nums[fast]; // fast 每次走一步
            slow = nums[slow]; // slow 每次走一步
        }

        return slow; // 返回相遇位置，即为重复的数字
    }
}

为什么这样做有效？
环的形成和相遇点
当数组中的元素值被视为链表中的指针时，重复的元素会导致链表中的环。快慢指针算法能够检测环的存在，并找到环的入口点。

假设环的入口点为 E，相遇点为 M，起点为 S，环的长度为 L。在相遇时，slow 走了 x 步，fast 走了 2x 步。因为 fast 比 slow 多走了一圈，所以有：

当 slow 走了 x 步到达相遇点 M 时，它实际上已经走了 k 圈环。

寻找环的入口点
为了找到环的入口点，我们将 fast 重置到数组的起点 S，然后同步移动 slow 和 fast。此时：

slow 从 M 开始，每次走一步。
fast 从 S 开始，每次走一步。
因为 slow 已经在环中走了 k 圈，而 fast 刚开始走，所以当两者相遇时，slow 走了 L - x 步，fast 走了 L - x 步，即两者在环的入口点 E 相遇。

总结
通过使用快慢指针技巧，我们可以高效地找到数组中的重复元素。这个算法的时间复杂度为 O(n)，空间复杂度为 O(1)，非常适合在不修改数组且只使用常数级别额外空间的情况下找到重复元素。
